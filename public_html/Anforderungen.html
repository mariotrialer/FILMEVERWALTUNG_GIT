Anforderungsanalyse

Phase 1


Funktion

* Eingabe eines Filmtitels, um ihn der Liste hinzuzufügen (z.B. Textfeld + Button)
* Eine Liste der vorhandenen Filme darstellen
* Filme sollen in der Liste geändert und gelöscht werden können
* Das Löschen eines Films soll durch eine Benutzereingabe bestätigt werden


Oberfläche

* Unterscheidung zwischen angemeldeten Benutzern und Gastbenutzern: nur angemeldete Benutzer dürfen die Daten ändern,
  d.h. Gastbenutzer sehen nicht das Eingabefeld und die Aktionen zum Ändern/Löschen eines
  Filmes (username/password kann zunächst hart kodiert sein)


Bemerkungen

* In dieser Phase ist es noch nicht nötig, dass die Filme in einer Datenbank gespeichert werden!
* Versuchen Sie, den Programmcode möglichst lesbar zu gestalten (sinnvolle Variablen- und Methodennamen (möglichst Englisch,
    einheitliche Formatierung, konsistenter Code-Stil)
* Beispiel eines JavaScript-Coding-Style-Guides: https://github.com/rwldrn/idiomatic.js/tree/master/translations/de_DE


Hinweise

* Sie können die Musterlösung zur Übung „Login“ als Ausgangsprojekt verwenden.
* Um beim Laden der Seite bereits ein paar Filme in der Liste zu zeigen, können Sie diese als „Beispiele” fest im HTML-Dokument vorgeben.


Phase 2


Funktionen

* Die Liste kann nach gesehenen/ungesehenen Filmen gefiltert werden
* Doppelte Filme oder Filme ohne Namen sollen vermieden werden
* Bei Eingabe eines ungültigen Titels (z.B. Titel ist leer oder besteht nur aus Leerzeichen, bereits vorhandener Titel) soll der Benutzer auf die
   ungültige Eingabe hingewiesen werden (z.B. Feld rot markieren mit einem Fehlertext unterhalb des Feldes)

Oberfläche

* Filme können als gesehen markiert werden
* Filme können bewertet werden (z.B. 1-5 Sterne)
* Die Liste der Filme kann auf- und absteigend nach Name und eigener Bewertung sortiert werden


Phase 3


Funktionen

Zu den Filmen in der Liste sollen zusätzliche Informationen aus einer externen Datenquelle eingebunden werden:
Als externe Datenquelle soll eine Film-API verwendet werden. Beispiele sind OMDb (keine Anmeldung notwendig) oder TMDb (Registrierung und API-key notwendig)
OMDb
TMDb (the movie database) (API-Dokumentation: http://www.themoviedb.org/documentation/api)
Sie können auch eine andere Filmdatenbank verwenden aber diese sollte eine API haben.

Oberfläche

* Benutzer klickt bei einem Film auf "Details"
* Im Hintergrund werden Daten über AJAX aus der externen API geladen
* Unterhalb des Films, als Dialog oder auf einem weiteren Screen wird folgendes angezeigt:
Bild vom Filmplakat, die Bewertung des Filmes, Erscheinungsjahr, Regisseur, Laufzeit, Genres

Hinweise

Wie behandeln Sie den Fall, dass ein Film in der API nicht gefunden wird oder dass es mehrere Filme mit dem Titel gibt?
